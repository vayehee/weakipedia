# Weakipedia Database Schema

These tables describe the current intended Weakipedia data architecture.

## Naming Rules

- `users` means Weakipedia signed-in users.
- `w_editors` means Wikipedia editor accounts.
- `user_*` tables store data owned by, addressed to, or subjectively created by Weakipedia users.
- `w_*` tables store objective Wikimedia-derived entities or analysis.
- `system_*` tables store operational system records.
- `target_*` tables store target-level data shared across entity types.

## Tables

| name | desc | api | fields | rules |
|---|---|---|---|---|
| `users` | Weakipedia signed-in user account | Telegram MTProto, IP geolocation, analytics, later email verification | `id`, `telegram_user_id`, `telegram_phone_hmac`, `telegram_username`, `display_name`, `photo_url`, `email`, `email_verified_at`, `phone_country`, `ip_country`, `analytics_country`, `status`, `created_at`, `last_login_at`, `updated_at` | Only for Weakipedia users. `telegram_user_id` is unique. `telegram_phone_hmac` is for privacy-preserving lookup, not display. Country fields are inferred signals and may disagree, so do not treat them as verified residence or citizenship. |
| `user_sessions` | Active Weakipedia login sessions | MTProto, internal auth | `id`, `user_id`, `session_token_hash`, `ip_address`, `user_agent`, `expires_at`, `revoked_at`, `created_at`, `last_seen_at` | `user_id` references `users.id`. Never store raw session tokens. |
| `user_login_verifications` | Telegram login verification attempts | MTProto | `id`, `phone_hmac`, `telegram_user_id`, `code_hash`, `status`, `attempt_count`, `ip_address`, `expires_at`, `created_at`, `verified_at` | Temporary login flow table. Keeps verification state out of `users`. Never store raw verification codes. |
| `user_email_verifications` | Email verification attempts | Email provider | `id`, `user_id`, `email`, `code_hash`, `status`, `attempt_count`, `expires_at`, `created_at`, `verified_at` | `user_id` references `users.id`. Never store raw email verification codes. |
| `user_telegram_messages` | Telegram messages sent or received by Weakipedia | Telegram MTProto | `id`, `user_id`, `phone_hmac`, `telegram_user_id`, `message_type`, `direction`, `status`, `telegram_message_id`, `payload_json`, `created_at`, `sent_at`, `failed_at` | `user_id` references `users.id` when known. `phone_hmac` is used before the person becomes a Weakipedia user. Use for login codes, invite links, alerts, system notices, and failures. |
| `user_events_audit` | User-related audit trail | Internal | `id`, `user_id`, `event_type`, `entity_type`, `entity_id`, `payload_json`, `ip_address`, `created_at` | Tracks important actions: login, invite, grant access, revoke access, subjective edit, alert change, and other security-relevant events. |
| `user_preferences` | Weakipedia user settings | Internal | `id`, `user_id`, `theme`, `language`, `timezone`, `notification_settings_json`, `dashboard_settings_json`, `created_at`, `updated_at` | Keeps settings out of `users`. |
| `projects` | User-created workspace/grouping of targets | Internal | `id`, `owner_user_id`, `name`, `description`, `visibility`, `status`, `created_at`, `updated_at` | `owner_user_id` references `users.id`. Projects can contain articles, Wikipedia editors, Wikidata items, and future target types. |
| `project_targets` | Targets included in a project | Internal | `id`, `project_id`, `target_id`, `added_by_user_id`, `created_at` | `project_id` references `projects.id`; `target_id` references `targets.id`; `added_by_user_id` references `users.id`; use `UNIQUE(project_id, target_id)`. |
| `project_members` | Broad project-level access for Weakipedia users | Internal | `id`, `project_id`, `user_id`, `role`, `invited_by_user_id`, `status`, `created_at`, `updated_at` | `role = owner/editor/viewer`; `status = invited/active/removed/declined`; use `UNIQUE(project_id, user_id)`. Subjective edits are still attributed to the real `users.id`. |
| `user_project_invitations` | Broad project invitations not limited to one panel | Internal, MTProto | `id`, `project_id`, `invited_by_user_id`, `intended_phone_hmac`, `role`, `token_hash`, `status`, `expires_at`, `created_at`, `redeemed_at`, `redeemed_by_user_id` | Use when inviting someone to a full project, unlike segmented `project_share_links`. |
| `project_share_links` | Generated segmented project access URL | Telegram MTProto, internal auth | `id`, `project_id`, `target_id`, `segment`, `created_by_user_id`, `intended_phone_hmac`, `token_hash`, `role`, `status`, `expires_at`, `created_at`, `redeemed_at`, `redeemed_by_user_id` | Pending invite intent. Never store the raw URL token. Valid only if authenticated Telegram phone matches `intended_phone_hmac`. Segment can be `overview`, `claims`, `sources`, `editors`, `traffic`, or `all`. |
| `project_panel_access` | Granted access to a specific project target segment | Internal | `id`, `project_id`, `target_id`, `segment`, `user_id`, `role`, `granted_by_user_id`, `source_share_link_id`, `status`, `created_at`, `updated_at` | Actual permission after successful redemption. Allows narrow access without full project membership. |
| `targets` | Dashboard subject, static or dynamic | Internal | `id`, `type`, `user_id`, `entity_type`, `entity_id`, `route`, `lang`, `title_slug`, `canonical_title`, `status`, `created_at`, `updated_at` | `type = static` for anonymous frozen dashboards, usually `user_id = null`; `type = dynamic` for signed-in Weakipedia users and should reference `users.id`. |
| `system_target_snapshots` | Frozen resolved state of a target at a point in time | Internal | `id`, `target_id`, `snapshot_type`, `snapshot_json`, `created_at` | System-owned. Useful for static dashboards and historical comparison. |
| `target_relationships` | Graph relationships between targets and related entities | Wikipedia parse, Wikidata, Wikinav, internal analysis | `id`, `from_target_id`, `to_target_id`, `relationship_type`, `entity_type`, `entity_id`, `weight`, `confidence`, `source_query_id`, `source_query_kind`, `created_at` | `from_target_id` references `targets.id`. `to_target_id` is used when the related item is also a target. `entity_type/entity_id` is used when the related item is not a target, such as source, editor, Wikidata item, claim, or project. |
| `static_builds` | One-time frozen build for anonymous/static targets | Internal | `id`, `target_id`, `started_at`, `finished_at`, `status`, `error_message` | Only for `targets.type = static`; usually one completed static build per static target. |
| `api_queries` | External API calls made during a static build | All external APIs | `id`, `static_build_id`, `source_type`, `request_url`, `http_status`, `response_json`, `response_hash`, `fetched_at`, `status`, `error_message` | Static-only. One row per external API query. Stores full raw JSON response. |
| `cron_api_queries` | Scheduled external API calls for dynamic signed-in dashboards | All recurring external APIs | `id`, `target_id`, `source_type`, `cadence`, `request_url`, `http_status`, `response_json`, `response_hash`, `last_fetched_at`, `next_fetch_at`, `status`, `error_message` | Dynamic-only. Used for hourly, daily, or monthly refreshes for signed-in Weakipedia users. |
| `system_background_jobs` | Async jobs and tasks | Internal | `id`, `job_type`, `target_id`, `user_id`, `status`, `payload_json`, `error_message`, `started_at`, `finished_at`, `created_at` | Handles builds, refreshes, parsing, alerts, Telegram sends, and email sends. |
| `system_rate_limits` | Abuse and rate-control records | Internal | `id`, `scope_type`, `scope_key`, `action`, `count`, `window_starts_at`, `window_ends_at`, `created_at`, `updated_at` | For login attempts, search, share redemption, API calls, alerts, and other bounded actions. |
| `w_articles` | Canonical Wikipedia article entity | Wikipedia `parse`, Wikipedia `query` | `id`, `lang`, `title_slug`, `canonical_title`, `canonical_url`, `wikidata_item_id`, `wikidata_qid`, `latest_revid`, `created_at`, `updated_at` | One row per language/title. Preserve Wikipedia slug style, underscores, accents, and special characters. `wikidata_item_id` references `wdata_items.id`; `wikidata_qid` is a denormalized convenience value. |
| `wdata_items` | Canonical Wikidata item entity | Wikidata `wbgetentities`, `wbsearchentities` | `id`, `qid`, `label`, `description`, `canonical_url`, `created_at`, `updated_at` | One row per Wikidata item. Used for direct Wikidata dashboards or linked article identity. |
| `w_editors` | Canonical Wikipedia editor account/entity | Wikipedia `allusers`, revisions API | `id`, `lang`, `editor_name`, `editor_page_url`, `edit_count`, `groups`, `registration_date`, `created_at`, `updated_at` | Use `editor`, not `user`, for Wikipedia accounts. This is objective and independent of Weakipedia account ownership. |
| `user_w_editor_accounts` | Wikipedia editor accounts claimed or controlled by a Weakipedia user | Wikipedia editor verification flow | `id`, `user_id`, `w_editor_id`, `claim_status`, `verified_at`, `verification_method`, `created_at`, `updated_at` | `user_id` references `users.id`; `w_editor_id` references `w_editors.id`; use `UNIQUE(user_id, w_editor_id)`. This is account control/claiming, not article-specific editor analysis. |
| `w_article_views` | Parsed Wikipedia article pageview data | Wikimedia Pageviews REST API | `id`, `target_id`, `article_id`, `source_query_id`, `source_query_kind`, `date`, `desktop_views`, `mobile_web_views`, `mobile_app_views`, `spider_views`, `automated_views`, `human_views`, `machine_access` | Objective data. `source_query_kind = api_queries` or `cron_api_queries`. Usually one row per date. `human_views = desktop_views + mobile_web_views + mobile_app_views`; `machine_access = spider_views + automated_views`. |
| `w_article_traffic` | Parsed article traffic in/out | Wikinav API | `id`, `article_id`, `source_query_id`, `source_query_kind`, `month`, `direction`, `title`, `views`, `title_type`, `url`, `created_at` | Objective article-centered Wikinav data. `month`, `title`, and `views` preserve Wikinav response values. `direction = in` for sources and `direction = out` for destinations. `title_type` is Weakipedia's mapped classification of Wikinav titles. `url` is nullable and derived only for normal Wikipedia pages. Use `UNIQUE(article_id, month, direction, title)`. |
| `g_trends` | Parsed Google Trends data | Google Trends API/provider | `id`, `target_id`, `source_query_id`, `source_query_kind`, `entity_type`, `entity_id`, `date`, `score`, `region`, `query` | Objective data. May be based on article title, Wikidata label, or Wikipedia editor name. |
| `g_news` | Parsed Google News data | Google News API/provider | `id`, `target_id`, `source_query_id`, `source_query_kind`, `entity_type`, `entity_id`, `published_at`, `title`, `source`, `url`, `snippet` | Objective data. |
| `w_article_revisions` | Objective Wikipedia revision records | Wikipedia revisions API | `id`, `target_id`, `article_id`, `source_query_id`, `source_query_kind`, `revision_id`, `parent_revision_id`, `editor_id`, `timestamp`, `comment`, `size_bytes`, `minor`, `created_at` | "What Wikipedia says happened." `editor_id` references `w_editors.id`. |
| `w_article_text_authorship` | Objective token or phrase-level authorship attribution for surviving article text | WikiWho API | `id`, `target_id`, `article_id`, `source_query_id`, `source_query_kind`, `revision_id`, `editor_id`, `token_text`, `token_start`, `token_end`, `introduced_revision_id`, `introduced_at`, `section_id`, `created_at` | "Who wrote this surviving text." `editor_id` references `w_editors.id`; `introduced_revision_id` references the revision where the text entered the article when available. |
| `w_article_sections` | Parsed article structure | Wikipedia `parse` | `id`, `target_id`, `article_id`, `source_query_id`, `source_query_kind`, `section_index`, `section_title`, `level`, `anchor`, `text_hash`, `created_at` | Useful for anchoring claims, sources, and edits to article structure. |
| `w_article_links` | Internal Wikipedia links extracted from article | Wikipedia `parse` | `id`, `target_id`, `article_id`, `source_query_id`, `source_query_kind`, `linked_lang`, `linked_title`, `linked_url`, `section_id`, `created_at` | Internal wiki graph and related-entity data. |
| `w_article_args` | Objective machine-extracted arguments from an article | Wikipedia `parse` plus Weakipedia analysis/LLM | `id`, `target_id`, `article_id`, `source_query_id`, `source_query_kind`, `argument_key`, `argument_text`, `section_title`, `source_span_start`, `source_span_end`, `confidence`, `extracted_at` | Objective/system-derived. One row per identifiable argument. |
| `user_w_article_args` | Subjective Weakipedia user layer over article arguments | Weakipedia user input | `id`, `user_id`, `w_article_arg_id`, `stance`, `suggested_rewording`, `suggested_source_url`, `note`, `created_at`, `updated_at` | `user_id` references `users.id`. Multiple Weakipedia users may disagree about the same objective argument. Usually `UNIQUE(user_id, w_article_arg_id)`. |
| `target_sources` | Canonical external web sources cited in Wikipedia articles or discovered around targets | Wikipedia `parse`, Google News/RSS, source fetcher, user paste | `id`, `canonical_url`, `original_url`, `url_hash`, `domain`, `registrable_domain`, `title`, `description`, `published_at`, `accessed_at`, `author`, `publisher`, `publication_name`, `language`, `archive_url`, `archive_date`, `doi`, `isbn`, `issn`, `pmid`, `page`, `pages`, `volume`, `issue`, `raw_metadata_json`, `full_text`, `full_text_added_by_user_id`, `full_text_added_at`, `created_at`, `updated_at` | One row per unique external source URL. No `source_type`, `country`, `quote`, or `raw_citation`; those belong in target/claim/news relationship tables because the same source can appear in different contexts. |
| `w_article_claims_sources` | Mapping between article claims and the sources used for them | Wikipedia `parse` plus Weakipedia analysis | `id`, `target_id`, `article_id`, `w_article_arg_id`, `target_source_id`, `citation_text`, `section_id`, `confidence`, `created_at` | Links one claim/argument to one external source. `w_article_arg_id` references `w_article_args.id`; `target_source_id` references `target_sources.id`. |
| `user_w_article_sources` | Subjective Weakipedia user layer over article sources | Weakipedia user input | `id`, `user_id`, `w_article_source_id`, `assessment`, `suggested_source_url`, `note`, `created_at`, `updated_at` | `user_id` references `users.id`. Usually `UNIQUE(user_id, w_article_source_id)`. |
| `w_article_editors` | Objective analysis of Wikipedia editors who edited a specific article | Wikipedia revisions API | `id`, `target_id`, `article_id`, `source_query_id`, `source_query_kind`, `editor_id`, `edit_count`, `first_edit_at`, `last_edit_at`, `revert_count`, `minor_edit_count` | `editor_id` references `w_editors.id`. This is article-specific editor activity, not Weakipedia user identity. |
| `user_w_article_editors` | Subjective Weakipedia user layer over article-editor analysis | Weakipedia user input | `id`, `user_id`, `w_article_editor_id`, `assessment`, `note`, `created_at`, `updated_at` | `user_id` references `users.id`. Usually `UNIQUE(user_id, w_article_editor_id)`. |
| `user_alerts` | User-defined alerts on target segments or metrics | Internal alert engine | `id`, `user_id`, `target_id`, `project_id`, `segment`, `metric`, `condition`, `threshold_value`, `threshold_unit`, `delivery_channel`, `status`, `last_triggered_at`, `created_at`, `updated_at` | `user_id` references `users.id`; `target_id` references `targets.id`; `project_id` is nullable and used when the alert belongs to a project context. |
| `user_alert_events` | Actual alert trigger history | Internal alert engine | `id`, `user_alert_id`, `target_id`, `metric`, `observed_value`, `event_payload`, `triggered_at`, `delivery_status`, `delivered_at` | Keeps alert definitions separate from alert occurrences. Useful for history, debugging, deduplication, and delivery tracking. |

## Deliberately Omitted
